# Log Rules

Ao analisar logs:

- leia a mensagem completa
- identifique arquivo, linha e comando
- diferencie sintoma de causa raiz
- não assuma causas sem evidência
- use a saída de testes como fonte principal de diagnóstico
- se houver múltiplos erros, priorize o primeiro erro relevante
- ao propor correção, diga quais arquivos precisam mudar
- registre o log em tools/codex/logs no formato `<dd-MM-yyyy_HH:mm:ss>_log_codex.log`
